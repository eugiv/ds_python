# Рекомендательная система для маркетплейса

Рекомендательная система предназначена для формирования рекомендаций **маркетинговой и SEO-командам** по продвижению товаров на маркетплейсе.  
Рекомендации строятся на уровне **товарных SKU** и основаны на анализе **пользовательских поисковых запросов** и сопутствующих метрик:

- позиция (место) в поисковой выдаче,
- количество уникальных пользователей по запросу,
- конверсия запроса и др.

---

## Цели системы

1. **Повышение конверсии из поиска**  
   Выявить ключевые поисковые фразы, влияющие на конверсию, и рекомендовать их к внедрению в карточки товаров.

2. **Оптимизация рекламных инвестиций**  
   Определить целесообразность вложения бюджета в продвижение конкретных SKU, особенно тех, что демонстрируют потенциал высокой конверсии по высокочастотным запросам.

3. **Поиск точек роста через схожие SKU**  
   Выявить SKU, имеющие общие поисковые фразы, и использовать успешные практики одного SKU для улучшения другого.

---

## Подходы к формированию рекомендаций

### 1. Baseline: Рекомендация эффективных запросов

Для каждого SKU формируется топ наиболее эффективных поисковых запросов (например, топ-5 или топ-10), которые:

- привели к продажам **или**
- имеют высокую конверсию просмотров **и**
- занимают относительно хорошую позицию в выдаче.

> **Суть действия**: рекомендует эффективные запросы для SEO и рекламы.

---

### 2. Content-Based Filtering (CBF)

Если два SKU схожи по семантике пользовательских запросов (например, на основе TF-IDF векторов и косинусного сходства), то успешные практики одного можно применить к другому.

Примеры использования:
- Для SKU с недостаточным объёмом данных — «подсмотреть» работающие фразы у похожего SKU.
- Дополнить карточку товара релевантными SEO-фразами из аналогов.

> **Суть действия**: находит SKU с похожими названиями / поисковыми запросами.

---

### 3. Collaborative Filtering (CF)

Если два SKU часто встречаются в одних и тех же поисковых запросах пользователей, они считаются поведенчески схожими.

> **Суть действия**: находит SKU, ведущие себя схоже в поиске.

---

### 4. Hybrid Model (CBF + CF)

Объединяет подходы Content-Based и Collaborative Filtering с весами:

- **60% — Content-Based Filtering**  
- **40% — Collaborative Filtering**

Такой гибрид позволяет находить SKU, которые одновременно:
- семантически близки по тексту,
- часто появляются в одних и тех же поисковых сессиях.

> **Суть действия**: улучшает качество рекомендаций за счёт комбинирования контентного и поведенческого сходства.  
> Позволяет переносить успешные SEO-фразы и рекламные стратегии между схожими SKU.
